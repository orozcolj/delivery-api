# Guía de Contribución

¡Gracias por tu interés en contribuir a este proyecto!

## Reglas básicas
- Usa ramas para nuevas funcionalidades o correcciones.
- Haz pull requests claros y descriptivos.
- Sigue el estándar PSR-12 para PHP y buenas prácticas de Laravel.
- Incluye pruebas automatizadas para nuevas funcionalidades.

## Estructura del proyecto
- `app/`: Lógica principal de la API y el frontend.
- `routes/`: Definición de rutas web y API.
- `database/`: Migraciones, seeders y factories.
- `resources/views/`: Vistas Blade del frontend.
- `public/`: Archivos públicos y punto de entrada.
- `tests/`: Pruebas unitarias y funcionales.

## Reporte de errores
- Usa Issues para reportar bugs o sugerencias.
- Incluye pasos para reproducir el problema y capturas si es posible.

## Documentación
- Actualiza `openapi.yaml` y el README si modificas endpoints o flujos importantes.
